    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-ruby-intro" class="anchor" href="#ruby-intro" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Ruby Intro</h1>

<p>This 3-part homework gives some basic practice in Ruby as well as
getting you accustomed to making testing a regular part of your workflow.</p>

<h2><a id="user-content-learning-goals" class="anchor" href="#learning-goals" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Learning Goals</h2>

<p>After completing this assignment, you will know how to:</p>

<ul>
<li>Write simple code that uses basic constructs in the Ruby language, including methods and arguments, conditionals, string and array manipulation, regular expressions, and basic object-oriented programming mechanisms</li>
<li>Understand the Ruby project conventions for where code files and test files are located in a project's directory hierarchy</li>
<li>Run individual tests or suites of tests using the RSpec unit testing tool</li>
<li>Understand the basic syntax of RSpec unit tests</li>
</ul>

<h2><a id="user-content-overview" class="anchor" href="#overview" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Overview</h2>

<p>The repo for this assigment follows a fairly standard Ruby convention for codebases: the code
files are stored in <code>lib/</code> and the test files are stored in <code>spec/</code>.
(We use the RSpec unit-testing framework; if we were using Ruby's default
framework, known as <code>Test::Unit</code>, the test files would be under
<code>test/</code>.)</p>

<p>We've placed "starter code" in <code>lib/ruby_intro.rb</code>; when you're all done, you
can submit this single file to the autograder.</p>

<p>However, you can test each of the 3 parts separately.  The files
<code>spec/part[123]_spec.rb</code> contain RSpec tests for each of the three
parts.  For example, to test your answers to Part 1, say <code>rspec
spec/part1_spec.rb</code>.  <code>rspec</code> with no arguments runs the tests in all
the files <code>spec/*_spec.rb</code>.</p>

<ul>
<li>The line numbers in the RSpec error report will
give you guidance as to which tests failed.  (You can check the <a href="http://rspec.info">RSpec
documentation</a> to see how the <code>.rspec</code> file can be
used to customize the output format.)</li>
</ul>

<p>To ensure you have the rspec gem installed you need bundler and can then 
run bundle install like so:</p>

<div class="highlight highlight-source-shell"><pre>$ gem install bundler
$ <span class="pl-c1">cd</span> hw-ruby-intro
$ bundle</pre></div>

<p>When the above completes successfully you'll have RSpec installed and can
run <code>rspec</code> from the command line to test your code.</p>
